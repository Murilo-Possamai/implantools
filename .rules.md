# Regras de Desenvolvimento do Projeto: implantools

Este arquivo define as diretrizes arquiteturais, a estrutura de pastas e as regras de comportamento da IA para o desenvolvimento do projeto `implantools`. Leia e aplique estas regras em todas as interações.

## 1. Escopo e Arquitetura do Projeto
- **Nome do Projeto**: implantools.
- **Natureza**: Ferramenta de utilidades com dupla interface (Web GUI via localhost e CLI - Command Line Interface).
- **Paridade Obrigatória (GUI = CLI)**: TODA E QUALQUER funcionalidade disponível como botão no menu Web localhost DEVE, obrigatoriamente, poder ser executada via linha de comando (ex: `--nome-do-comando`). Não crie funções exclusivas para a interface Web que não possam ser chamadas pelo terminal.

## 2. Estrutura de Diretórios e Organização do Código
- **CLI Entrypoint (`main.py`)**: Localizado na raiz, é o console/terminal do sistema. Ele é responsável pelo parsing dos comandos de linha e por executar as ações. Se executado com a flag `--gui`, ele deve delegar a execução para o `web.py`. Nenhuma lógica de negócio deve ser escrita diretamente nele.
- **Web Entrypoint (`web.py`)**: Localizado na raiz, contém exclusivamente a configuração e inicialização do servidor web (localhost) e as rotas da interface gráfica. Ele consome as mesmas funções da pasta `tools/` que o `main.py` consome.
- **Diretório `tools/`**: Todas as ações, ferramentas e funcionalidades específicas DEVEM ser modularizadas dentro da pasta `tools/`.
- **Estrutura das Tools**: Cada nova ferramenta deve residir em seu próprio subdiretório (Ex: `tools/nome_da_tool/`).
- **Separação de Responsabilidades**: Dentro de cada ferramenta, a lógica deve ser subdividida. Utilize pastas como `tools/nome_da_tool/actions/` para a execução das tarefas e `tools/nome_da_tool/controller/` para gerenciar o fluxo de dados.

## 3. Interface de Linha de Comando (CLI) e Ajuda
- **Geração Automática**: O mapeamento de funções para a CLI no `main.py` deve ser feito de forma automatizada/centralizada, lendo as ferramentas disponíveis na pasta `tools/`.
- **Auto-Help (`--help`)**: Ao adicionar uma nova opção/ferramenta no sistema, o comando `--help` deve ser atualizado e implementado automaticamente para refletir as novas flags de comando geradas.
- **Flag GUI**: O comando `python main.py --gui` (ou a respectiva flag no binário compilado) é o gatilho oficial para iniciar a interface web.

## 4. Interface Web (GUI Localhost)
- **Acesso**: A interface gráfica rodará exclusivamente no navegador via localhost, inicializada pelo `web.py`.
- **Menu Central**: O sistema deve possuir um menu principal onde cada funcionalidade (importada de `tools/`) é acessível por um botão claro e direto.
- **Estilização Mínima**: A interface Web deve ter uma estilização básica, limpa e funcional (minimalista). O foco é na usabilidade e legibilidade.
- **Visibilidade de Funcionalidades**: Toda e qualquer funcionalidade (formulários de input, campos, resultados) SÓ DEVE aparecer na interface APÓS o usuário clicar no seu respectivo botão no menu principal. A tela inicial (home) deve ser limpa e conter apenas os botões de navegação.
- **Botão de Cópia**: Sempre que um bloco de código ou texto for gerado como resultado de uma ação, um botão "Copiar" deve ser exibido acima dele para facilitar a cópia do conteúdo para a área de transferência.

## 5. Gerenciamento de Dependências
- **Atualização do `requirements.txt`**: Toda e qualquer nova biblioteca externa ou pacote de terceiros que for utilizado no código DEVE ser imediatamente e obrigatoriamente adicionado ao arquivo `requirements.txt` na raiz do projeto.

## 6. Regras Estritas de Comportamento da IA
- **Acesso a Arquivos (REGRA DE OURO)**: Se durante a resolução de um problema ou criação de uma feature você (a IA) precisar analisar, modificar ou interagir com um arquivo que **não está explicitamente fornecido no seu contexto atual**, VOCÊ DEVE PARAR. Não invente, não alucine código e não faça suposições. **Solicite explicitamente que o usuário forneça o conteúdo do arquivo faltante antes de continuar a resposta.**
- **Código Limpo**: Mantenha o código estritamente modular. O `main.py` e o `web.py` atuam apenas como pontes (interfaces) para a verdadeira lógica de negócio que reside na pasta `tools/`.
